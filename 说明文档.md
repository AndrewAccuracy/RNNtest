# RNN 延迟记忆实验

## 项目目标（两阶段叙事）

实验拆成两段，分别对应演示里常见的两层问题：

1. **阶段一：RNN 相对「以前的」建模范式**  
   对比 **固定窗口（History-MLP）**、**全长展平 MLP（Flat-MLP）** 与 **RNN**。  
   - 固定窗口往往在长 gap 下**根本看不到**首 token，体现「局部历史」的局限。  
   - Flat-MLP 与 RNN **看见同一串符号**，但不共享跨时间的权重；适合讲 **归纳偏置** 与 **参数量随长度变化**（展平输入维随序列长度增长）。

2. **阶段二：LSTM 相对「普通 RNN」**  
   对比 **RNN** 与 **LSTM**：在长依赖区间，普通 RNN 往往更不稳定，LSTM 常更稳——对应你关心的 **门控与长程记忆**。

## 任务设计

每条输入序列长这样：

```
A c d e f g c d e f g ... ?
```

- **第一个符号**（`A` 或 `B`）是唯一有用的信息  
- **中间**是一堆随机干扰符（从 `c d e f g` 中随机抽取）  
- **最后一个** `?` 代表「请预测开头是 A 还是 B」  

这是一个**二分类任务**。干扰段越长（gap 越大），越考验模型能否把信息从序列首端保留到末端。

默认在内存中按规则随机生成；也可用 `--export-data` 落盘后再用 `--data-dir` 固定数据做复现或答辩展示。

## 对比的四个模型

| 模型 | 代码类 / 名称 | 要点 |
|------|----------------|------|
| **History-MLP** | `FixedWindowMLP` | 只看末尾 `history_window` 个 token（默认 5），模拟固定窗口历史模型 |
| **Flat-MLP** | `FlatSequenceMLP` | 全长 one-hot **展平** 后接 MLP，**无**时间递归；与 RNN 输入信息集对齐，结构不同 |
| **RNN** | `RNNClassifier` | 逐步读入，隐状态沿时间传递 |
| **LSTM** | `LSTMClassifier` | 门控记忆，长序列上常更稳 |

## History-MLP 与 Flat-MLP 在代码里是什么

- **History-MLP**：`FixedWindowMLP`，见 `experiment.py` 中 `build_model(..., "History-MLP")`。  
- **Flat-MLP**：`FlatSequenceMLP`，每个 gap 下序列长度固定为 `gap_length + 2`，输入维为 `len(VOCAB) * (gap_length + 2)`。

默认 `gap` 较大时，首 token 落在固定窗口之外，History-MLP 的准确率会接近随机；Flat-MLP 与 RNN/LSTM 则仍**能**利用首 token 的信息（是否学得好另说）。

## 实验流程

对每个 **gap**，对四个模型各跑多个 **seed**，汇总均值与标准差；再按阶段拆成两套图与指标。

```
对每个 gap：
  对 History-MLP, Flat-MLP, RNN, LSTM：
    对每个 seed：
      → 构造训练/测试集（内存 或 --data-dir）
      → 训练若干 epoch
      → 记录测试准确率
  → 多 seed 聚合
→ 输出 phase1（History-MLP、Flat-MLP、RNN）、phase2（RNN、LSTM）及四模型总览图
```

### 为什么要多个随机种子（seed）

训练受初始化、打乱顺序等随机性影响；多 seed 取均值与方差，结论更稳。**注意**：若使用 `--data-dir`，各 seed 共用同一套落盘数据，方差主要来自初始化与 batch 顺序，与默认「每 seed 重采样数据」不同。

### 数据落盘（可选）

- **`--export-data`**：在训练前写入 `data/delayed_memory/gap_XXX/train.jsonl` 与 `test.jsonl`（默认根目录可用 `--data-export-root` 修改）。生成规则与内存版一致：训练用 `seeds[0] + 100`、测试用 `seeds[0] + 200` 的随机种子。  
- **`--data-dir PATH`**：从该路径按 gap 读入上述 JSONL，不再为每个 seed 重新采样数据。

每行 JSON 包含 `token_ids`、`tokens`（人类可读）、`label`。

## 常见问题：图里「看不出效果」、RNN 和 LSTM 叠在一起

- **原因**：在 **gap 较小**（例如只有 5 和 10）时，延迟记忆任务对 RNN/LSTM 都很容易，测试准确率都会接近 **100%**，Phase2 曲线会重叠，**不是模型坏了，也不是数据错了**。
- **怎么办**：拉大依赖长度，让长程记忆变难。请使用完整 gap 列表，例如：

```bash
python experiment.py --preset slides
```

等价于 `--gaps 5 10 18 20 22 26 30 50 100` 且 `--seeds 1 2 3 4 5`（最大 gap 为 100，便于看极长依赖）。训练前终端会打印当前 `gap_lengths`；若 `max(gap) < 18`，也会给出提示。

快速冒烟（仅验证能跑通）：

```bash
python experiment.py --preset quick --epochs 2
```

## 如何运行

```bash
source .venv/bin/activate
python experiment.py
```

答辩/正式出图推荐：

```bash
python experiment.py --preset slides
```

默认训练 **30** 个 epoch（可用 `--epochs` 修改）。三张「最长 gap」训练损失曲线图默认只绘制 **从第 12 个 epoch 起**（`--training-loss-epoch-start`，减轻前面平台期）；其中 **`training_loss_gap_max.png` 不含 Flat-MLP**，以免其很快收敛到 0、把纵轴拉得看不出 RNN/LSTM 的差异。

导出数据并训练（示例）：

```bash
python experiment.py --export-data --data-export-root data/delayed_memory
```

仅用落盘数据训练：

```bash
python experiment.py --data-dir data/delayed_memory
```
