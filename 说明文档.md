# RNN 延迟记忆实验

<p align="center"><sub>可复现的 PyTorch 教学基准：讲清 <b>为什么需要循环</b> 与 <b>为什么需要 LSTM</b> —— 为汇报与课程设计，而非刷榜。</sub></p>

> **短依赖好学，长依赖难扛；循环在网络间共享权重；LSTM 用门控决定记什么。** 本仓库在合成延迟记忆任务上跑两条对照叙事：阶段一用固定窗口 MLP、全长展平 MLP 对照 RNN；阶段二在 gap 拉长时用普通 RNN 对照 LSTM。一条命令即可得到置信带曲线、热力图、JSON 指标与 copy 任务学习曲线，可直接插入幻灯片。

## 为什么做这个项目

循环网络「好讲难证」。只在 MNIST 上报一个准确率，说明不了：

| 没有可控任务时 | 容易漏掉什么 |
|---|---|
| 「RNN 能处理序列」 | **哪一段**信息重要、**隔多远**才难 |
| 「LSTM 缓解梯度消失」 | 普通 RNN 明显失败、LSTM 仍能学的 **可见区间** |
| 「换个更大的 MLP 就行」 | 相同 token、不同 **归纳偏置**——共享权重 vs 展平 |

**本项目就是那个可控任务：** 合成序列、可扫 gap、多随机种子、按阶段拆图——十五分钟的汇报能展示 **机制**，而不是背结论。

技术栈很克制：

- **PyTorch** —— [`experiment.py`](experiment.py) 里 `nn.RNN` / `nn.LSTM` 与两个非循环基线。
- **可选 JSONL** —— `--export-data` / `--data-dir` 固定数据，便于答辩复现。
- **Copy 记忆跟进实验** —— [`experiment_lstm_copy_task.py`](experiment_lstm_copy_task.py) 里更干净的 RNN vs LSTM 故事。

## 任务设计

每条序列形如（阶段二使用前 `k` 个 token 作为模式前缀，默认 `k=2` → 4 类）：

```text
A B c d e f g ... ?
A A c d e f g ... ?
B A c d e f g ... ?
```

- **前 `k` 个 token** 编码类别（模式）。
- **中间** 为干扰符 `{c,d,e,f,g}` 的随机序列。
- 末尾 **`?`** 要求预测模式类别。

**gap 越大** → 干扰段越长 → 长程信用分配越难。汇报里拧的就是这个旋钮。

## 对比模型

| 模型 | 在叙事里的角色 |
|---|---|
| **History-MLP**（`FixedWindowMLP`） | 只看末尾 `history_window` 个 token（默认 5）——「旧式」固定窗口基线。 |
| **Flat-MLP**（`FlatSequenceMLP`） | 全长 one-hot 展平后接 MLP；与 RNN **看见相同符号**，但 **不** 跨时间共享权重，输入维随长度增长。 |
| **RNN**（`RNNClassifier`） | 逐步读入，隐状态向前传递。 |
| **LSTM**（`LSTMClassifier`） | 门控记忆——阶段二在长 gap 下的主角。 |

### 代码里两个 MLP 分别是什么

- **History-MLP**：`experiment.py` 中 `build_model(..., "History-MLP")` → `FixedWindowMLP`。
- **Flat-MLP**：`FlatSequenceMLP`；每个 gap 下序列长 `gap_length + 2`，输入维 `len(VOCAB) * (gap_length + 2)`。

gap 较大时，首 token 常落在固定窗口之外，History-MLP 准确率接近随机；Flat-MLP 与 RNN/LSTM **仍能** 利用首 token（能否学好另说）。

## 快速开始

```bash
git clone git@github.com:AndrewAccuracy/RNNtest.git
cd RNNtest
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

**默认两阶段实验**（写入 `outputs/`）：

```bash
python experiment.py
```

**汇报 / 正式出图推荐**（完整 gap + 5 个 seed）：

```bash
python experiment.py --preset slides
```

**快速冒烟**（验证能跑通）：

```bash
python experiment.py --preset quick --epochs 2
```

**自定义扫描**：

```bash
python experiment.py \
  --epochs 20 \
  --hidden-size 16 \
  --gaps 5 10 18 20 22 26 30 50 100 \
  --seeds 1 2 3 4 5 \
  --phase2-prefix-len 2 \
  --match-lstm-params
```

> 若只用较小 gap（如 5、10），RNN 与 LSTM 往往都接近 100%——曲线重叠是 **预期现象**，不是 bug。请用 `--preset slides` 或 gap ≥ 18 拉开阶段二。

### 实验二 — Copy 任务（RNN vs LSTM）

更直观的「为什么 LSTM」，含 token 级指标：

```bash
python experiment_lstm_copy_task.py \
  --gaps 15 18 \
  --seeds 1 2 3 \
  --memory-length 3 \
  --train-samples 1800 \
  --test-samples 400 \
  --batch-size 64 \
  --epochs 60 \
  --embedding-dim 32 \
  --hidden-size 128 \
  --num-symbols 4 \
  --token-threshold 0.75 \
  --output-dir outputs_exp2_copy_task_final
```

### 可选：导出 / 读取固定数据

导出 JSONL（随机规则与内存生成一致）：

```bash
python experiment.py --export-data
```

从磁盘训练（各 seed 共用同一套文件——方差主要来自初始化与 batch 顺序）：

```bash
python experiment.py --data-dir data/delayed_memory
```

每行 JSON 含 `token_ids`、`tokens`、`label`。

## 六条设计原则

### 1 · 两阶段，一个仓库。

阶段一回答「为什么不只用 MLP？」阶段二回答「为什么不只用 RNN？」`phase1_*` / `phase2_*` 图与幻灯片章节一一对应。

### 2 · Gap 是自变量。

其余条件尽量固定，只让 **依赖长度** 在横轴上移动——这样才是「展示长程依赖」，而不是空讲。

### 3 · Seed 是一等公民。

默认五个 seed → 均值曲线 + 变异性带。不能靠一次幸运初始化下结论。

### 4 · 基线公平但不同。

Flat-MLP 与 RNN 看见同一组 token；History-MLP 则否。对比的是 **归纳偏置**，不是故意刁难。

### 5 · 不止看准确率。

临界 gap、达到损失阈值的轮数、参数量匹配表，方便答辩 Q&A。

### 6 · 产出物就是交付物。

目标不是 SOTA，而是 **PNG + JSON + summary.txt**，插进 Keynote / PowerPoint 不用再画第二遍。

## 常见问题

**图里「看不出效果」、RNN 和 LSTM 叠在一起？**

- **原因：** gap 较小时（如只有 5、10），任务对两者都太容易，准确率都接近 100%，阶段二曲线重叠——**不是模型坏了**。
- **做法：** 使用 `python experiment.py --preset slides`（含 gap 至 100）。若 `max(gap) < 18`，终端会提示。

**多 seed 是干什么的？**

训练受初始化、打乱等影响；多 seed 看均值与波动。使用 `--data-dir` 时各 seed 共用同一套落盘数据，波动主要来自初始化与 batch 顺序。
